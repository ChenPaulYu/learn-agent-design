訓練怎麼打分 — Cross-Entropy 與 Teacher Forcing

模型怎麼從「猜下一個字」練成聰明的東西?這頁講訓練迴路本身。 對牌庫打分這件事,訓練時是 四個位置同時攤開來算的(teacher forcing),打分打得準不準,則靠 cross-entropy 變成一個可以拿去調參數的數字;最後也誠實揭露這套訓練方式的代價——exposure bias。

圖 A · Teacher Forcing
四個位置同時攤開,每個都餵「真實歷史」

示範句「今天天氣很好」拆成四張字牌:今天 / 天氣 / 很 / 好。因為整句正確答案本來就是 訓練語料本身,每個位置該餵的歷史在資料寫死那一刻就已經確定——不用等前一個位置先猜完, 四個位置可以同時算、同時打分。點下面任一張卡片看細節。

跟「整排一起看」卡榫卡準 — 整排一起看(self-attention)提供 「同時處理每個位置」的能力,teacher forcing 提供「每個位置的輸入不需要等前一位置生成完」 的前提,兩者疊在一起,訓練才能被推到大規模、平行運算。
位置 3 看到的是真實答案 — 就算位置 2 那一次模型自己猜錯了, 位置 3 拿到的輸入依然是真實的「今天、天氣」,不會被前一格的錯誤污染。
圖 B · Cross-Entropy Loss
打分打得準不準,怎麼變成一個要縮小的數字

每個位置模型都會對牌庫打分—— 給正確答案那張牌一個機率。Cross-entropy loss = -log(正確答案的機率):機率越低, loss 越高,而且是非線性放大的——越接近「完全猜錯」,懲罰越不成比例地重。 點下面的長條看每個位置當初打了多少分、扣了多少分(機率與 loss 皆為教學示意,非真實模型輸出)。

非線性怎麼看出來 — 機率從 0.9 掉到 0.5,loss 只增加一點; 但機率從 0.1 掉到 0.01,loss 會飆升得快很多。這逼著模型「有把握」的地方要真的有把握, 不能含糊地把機率平均攤開。
整句 loss 怎麼算 — 把四個位置各自的 loss 取平均,就是這句話 (或這個訓練批次)要拿去反向傳播、調整參數的數字——teacher forcing 負責「每個位置該餵 什麼」,cross-entropy 負責「猜測該怎麼變成數字」,兩者合起來才是完整的訓練迴路。
圖 C · Exposure Bias
誠實揭露的限制 — 訓練看真實歷史,推論吃自己接的

Teacher forcing 有一個結構性的代價:訓練時模型永遠看真實歷史,推論時卻沒有真實歷史可 餵——推論時每一步的「歷史」都是自己剛剛接出來的東西,一旦某一步接錯,下一步就是在一段 訓練時從沒見過的「歪掉的歷史」上運作。切換下面的模式,看同一組位置卡片怎麼跟著變。

示意腳本,非真實模型輸出 — 「沒有 teacher forcing」那組猜測 是為了說明「錯誤一步步累積放大」手編的教學劇本,不是任何真實模型的推論紀錄。
現代 LLM 幾乎不緩解這個代價 — GPT 系列一路到現在訓練時幾乎 清一色只用純 teacher forcing,不套用 scheduled sampling 這類緩解手法——換取可平行化訓練 的效率,是已知但被業界普遍接受的權衡,不是已解決的問題。
