換成 Diffusion,Tool Use 這些 Agent 技巧還撐得住嗎?

分層看差很多。Skills 這種「靠 context 內容驅動」的技巧,幾乎不受影響。Tool use 這種「需要中途暫停、等外部結果、再繼續」的迴圈,實測下來確實會出系統性問題——不是空想的 架構摩擦。深挖下去,真正的分界線不是「是不是 diffusion」,是更根本的 autoregressive(AR)vs non-autoregressive(NAR):AR 的生成順序跟真實世界資訊到達 的順序同構,這是它天生適合 agent 的根本原因。

區塊 1 · 先分兩層
哪些技巧只是「靠 context 驅動」,哪些需要「中途暫停」

不管生成機制是接龍還是去噪,「context 裡放了什麼會影響輸出」是兩種架構共通的基礎能力; 但「講到一半先停下來聽結果,再接著講」就不是所有架構都天生會的事。點卡片看差在哪。

區塊 3 · 更根本的分界線
不是「是不是 diffusion」,是 AR vs NAR

Diffusion 只是 NAR(non-autoregressive)陣營裡目前最紅的一種具體做法,問題的根源是 AR/NAR 這條軸。這件事有比 diffusion LM 早了快十年的歷史證據: Gu et al., 2018, Non-Autoregressive Neural Machine Translation(ICLR 2018)撞過一模一樣的牆,叫 multimodality problem—— 「Thank you very much.」翻成德文,「Danke schön.」跟「Vielen Dank.」都對,但如果各個 位置獨立平行決定,模型可能把兩種正確答案的碎片混在一起,吐出「Danke Dank.」這種四不像。 下面自己動手生成看看,AR 跟 NAR 差在哪一步。

區塊 4 · 根因
為什麼 AR 天生適合 agent:跟真實世界因果結構同構

打個比方:你問朋友「猜蘋果現在股價多少」,他可以隨口猜;但如果他要「真的查一下」,查詢 動作必須先做、結果必須先出來,他才能根據真實數字繼續講——他不可能在結果出來前就把後面 整段話定案。這正是 AR 的結構:每個字定案完,下一個字才輪到,跟「每個動作做完才知道 結果,才能決定下一步」是同一種時間順序。

區塊 6 · 修法
Block-wise diffusion 借回「定案」性質——但不是免費午餐

業界的答案不是放棄 diffusion,是把輸出切成一塊一塊(block-wise / semi-autoregressive diffusion,BD3-LM、Fast-dLLM v2、LLaDA 2.0 是具體案例):block 跟 block 之間嚴格 接龍(前一個 block 定案了,後一個才開始),block 內部才做平行去噪。拖動下面的滑桿, 看 block 大小怎麼在「安全」跟「平行」之間拉扯。

不是切一刀就搞定的開關 — block 切太小,保留局部條件依賴, 但要跑很多輪去噪;block 切太大,平行度看起來變高,但容易做出過早的決定、快取一致性 也變差。
活躍在解的問題 — 最新的做法(AdaBlock-dLLM、CtrlDiff)已經 把 block size 從訓練前定死的超參數,變成推論當下依樣本動態決定的問題,代表「切多大、 在哪裡切」仍沒有標準答案。
區塊 7 · 容易混在一起,但其實獨立
「Transformer vs Mamba」跟「AR vs NAR」是兩條互相獨立的軸

前者問「怎麼算出每個位置的表示」,後者問「輸出的各個位置,決定的順序/依賴關係長什麼樣」。 實務上兩兩都能組合——真正決定「tool use 好不好接」的是 AR/NAR 這條軸,不是 Transformer/Mamba 那條軸。

Transformer(attention)
Mamba / RWKV(狀態機)
AR
多數主流 LLM(GPT 系列)——接龍生成,用 attention 算每個位置的表示。
Mamba 語言模型——一樣是接龍生成,只是內部機制不是 attention。
NAR
多數 diffusion LM——內部其實也是拿 Transformer 的區塊在算,只是拿掉 causal mask、 改成雙向 + 迭代去噪。
筆記未點出具體案例——理論上兩軸可以自由組合,但這格不是本篇重點。
後續修正 — 上面「拿掉 causal mask、改成雙向」這個細節, 後來在 llm-primitives.md 用第一原理重新檢驗時發現不能輕輕帶過——causal mask 存不存在,其實是被解碼順序(AR/NAR) 決定的,不是架構軸自己能選的自由度。「混合機制(attention vs 狀態機)」才是真正 獨立於 AR/NAR 的架構原子,細節見該篇。

也不是所有 agent 相關能力都掛在同一條軸上——in-context learning 就是一個反例,見 in-context-learning-under-diffusion.md。
