# 位置編碼:整排一起看天生不分順序,怎麼補上「第幾個」

> TL;DR — [`transformer-self-attention.md`](2026-07-04-transformer-self-attention.md) 講的
> 「整排一起看」有一個天生的破洞:算關聯只看每張字牌牌櫃收納位置查出來的向量,完全不看這張牌
> 排在隊伍第幾個——把一句話的字牌打亂重排,算出來的關聯權重理論上不會變。原始論文
> ([Vaswani et al., 2017](https://arxiv.org/abs/1706.03762)) 的補法是**正弦波位置編碼**:
> 每個位置疊加一組固定、不學習的波紋數值。現在主流的補法是 **RoPE**
> ([Su et al., 2021](https://arxiv.org/abs/2104.09864)):不是疊加,而是把 Query/Key 向量按
> 位置比例**旋轉**一個角度,讓兩張牌的關聯分數天生只跟它們**相隔多遠**掛勾,不是各自站在隊伍
> 第幾號。

## 問題本身:整排一起看,天生分不出順序

先精確講一次「整排一起看」實際在算什麼。每張字牌會先各自被投影成三個向量:Query(我在找什麼)、
Key(我能提供什麼)、Value(我實際帶的內容)。算某張牌跟另一張牌的關聯分數,是拿**這兩張牌的
向量**做內積——僅此而已。

問題就在這裡:**這三個投影只吃牌櫃收納位置查出來的那條向量本身,不吃這張牌排在隊伍第幾個**。
把一句話「貓 追 老鼠」跟打亂後的「老鼠 追 貓」拿去算,如果不做任何額外處理,每張牌的 Query/Key/
Value 向量完全沒變(「貓」的向量還是那條「貓」的向量,跟它排第一還是排第三無關),兩兩之間算出
來的關聯權重也就跟著完全沒變——模型看到的是一袋牌的集合,而不是一列有順序的牌。這在數學上叫
**排列不變性(permutation invariance)**。

對照一下前一篇提過的「傳話遊戲」(RNN):傳話遊戲的傳遞方式**本身就內建了順序**——誰先講、
誰後講、傳了幾手,順序資訊天生就編在「傳話的過程」裡。整排一起看拿掉了排隊,換來平行運算的
好處,卻也連帶把「順序」這個資訊一起拿掉了——這不是一個好巧不巧的小疏忽,是這個機制的設計代價,
需要另外找辦法補回來。

而語言幾乎處處依賴順序:「貓追老鼠」跟「老鼠追貓」用的是同一批字牌,意思卻完全相反;所以這個
洞補不上,整排一起看這個機制就不能用在語言上。

## 原始解法:正弦波位置編碼(sinusoidal positional encoding)

[Vaswani et al., 2017](https://arxiv.org/abs/1706.03762) 的原始論文開的藥方是:**在每張牌的
向量上,疊加一組跟這張牌所在位置對應、事先算好、不需要訓練的固定數值**。

具體公式(`pos` 是位置、`i` 是向量裡第幾維、`d_model` 是向量總維度):

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

拆解成白話:向量的每一對維度,對應一個不同頻率的正弦/餘弦波——維度愈靠前,波長愈短(位置變
一格,數值變化很快);維度愈靠後,波長愈長(位置變很多格,數值才緩慢變化。把這一整組「不同
頻率的波」在某個位置上的數值算出來,疊加到那個位置的牌向量上,牌向量就從此帶著「我在第幾個」
的印記,而且不同頻率疊在一起讓模型有機會透過三角函數的性質,從兩個位置的編碼推算出它們的**相對
距離**,不只是各自的絕對位置。

這個做法有兩個特徵值得記住:

- **固定,不是學出來的**——這組波紋數值在訓練前就用公式算死了,不隨梯度更新;跟前一篇提過的
  「牌櫃收納位置」(embedding table)不一樣,牌櫃收納位置是訓練出來的參數,位置編碼不是。
- **用「疊加」的方式接進去**——直接把位置編碼的數值加到牌的向量上,牌向量從此同時帶著「這張
  牌的內容」跟「這張牌在第幾個」兩份資訊,混在同一條向量裡。

## 現代主流解法:RoPE——不是疊加,是「轉」

正弦波位置編碼管用,但混合「內容資訊」跟「位置資訊」在同一條向量裡,對「這兩張牌隔多遠」這件事
只是间接推得出來,不是算式天生保證的。[Su et al., 2021, *RoFormer*](https://arxiv.org/abs/2104.09864)
提出的 **RoPE(Rotary Position Embedding,旋轉位置編碼)** 換了一個思路:**不疊加,而是把
Query 跟 Key 向量按照位置比例「旋轉」一個角度**。

具體機制:把 Query/Key 向量兩兩一組看成平面上的座標點,每個位置對應一個旋轉角度(位置愈後面,
轉得愈多),讀牌前先把每張牌的 Query 向量、Key 向量各自按自己的位置轉過那個角度,再拿去做內積
算關聯分數。旋轉是一種等距變換的性質,使得**兩個轉過的向量做內積,結果只跟它們「轉了幾度的
差」有關,不跟它們各自轉了幾度的絕對值有關**——換算回位置,就是:兩張牌的關聯分數天生只跟它們
**相隔幾個位置**掛勾,不是各自排在隊伍第幾號。把整句話往右平移十個位置,每張牌之間的相對距離
沒變,算出來的關聯權重也不會變,這是正弦波疊加法沒有直接保證的一個乾淨性質。

RoPE 現在是主流開源大模型的預設做法——LLaMA、GPT-NeoX、PaLM 一類的模型都採用 RoPE,而不是
原始論文的疊加式正弦波編碼。

**再提一句 ALiBi,不展開。** [Press et al., 2021](https://arxiv.org/abs/2108.12409) 提出的
**ALiBi(Attention with Linear Biases)** 是另一種思路:完全不對牌的向量動手腳,而是在算完
關聯分數之後,直接按兩張牌隔多遠**扣一個跟距離成正比的固定懲罰值**——愈遠的牌,關聯分數被扣
愈多分。跟 RoPE 一樣是「天生綁定相對距離」的設計,但做法更簡單粗暴,論文的賣點主要是訓練用
短句子、推論時能撐到更長句子(它的論文名稱就叫「Train Short, Test Long」)。

## 三種做法放在一起比

| 做法 | 位置資訊接在哪 | 是否學習 | 天生綁定相對距離? |
| --- | --- | --- | --- |
| 不做任何處理 | 沒有 | — | 否(排列不變,問題本身) |
| 正弦波位置編碼 | 疊加到牌的向量上 | 否(公式算死) | 間接可推,非算式保證 |
| RoPE | 旋轉 Query/Key 向量 | 否(公式算死) | 是(內積結果天生只看差) |
| ALiBi | 扣在算完的關聯分數上 | 否(公式算死) | 是(懲罰值直接是距離的函數) |

## 容易被忽略的地方

- **位置編碼解的是「整排一起看」的副作用,不是額外加分的功能**——沒有它,整排一起看這個機制
  在語言任務上根本不成立,不是「加了更好、不加也還行」的選配。
- **三種做法都不是靠訓練學出「位置」這件事本身**——正弦波跟 RoPE 的角度/波形都是公式先算死
  的,模型學的是怎麼**運用**這份已經算好的位置訊號,不是從頭學什麼叫「第一個、第二個」。
- **「相對距離」比「絕對位置」更好用的原因**:一句話裡「這兩個字隔三個字」這件事,不管這句話
  出現在文件的開頭還是第 500 個字之後都一樣重要;死記「這是第 500 個字」對理解語意本身沒什麼
  用處,這是 RoPE、ALiBi 都朝相對距離設計的共同動機。
