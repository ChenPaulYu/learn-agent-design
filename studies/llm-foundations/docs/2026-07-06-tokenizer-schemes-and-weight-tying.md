# 造牌庫的兩種哲學,以及輸入輸出兩張表為什麼常常是同一張

> TL;DR — [`tokenizer-bpe.md`](2026-07-04-tokenizer-bpe.md) 講的 BPE 是「合併」哲學:從最細
> 字元開始,不斷把最常黏在一起的兩塊焊起來,牌庫是**長出來**的。另一個門派 **Unigram**(常見
> 於 SentencePiece 工具包)反過來做「篩選」:先列一大堆候選子詞,用機率模型幫每個候選打分,
> 反覆砍掉貢獻最小的,直到剩下目標張數——牌庫是**篩出來**的,而且同一個詞常常有好幾種合法切
> 法、各自帶機率。另外一個常被忽略的細節:模型輸入端「查牌櫃收納位置」的表,跟輸出端「對牌庫
> 打分」的表,很多模型是**同一份參數**(互為轉置)——這叫 weight tying,但不是所有模型都這樣做。

## 複習:BPE 是「合併」哲學

[`tokenizer-bpe.md`](2026-07-04-tokenizer-bpe.md) 已經走過一次:從最細的字元(或位元組)開始,
不斷數「相鄰配對出現次數」,把最高分的那對焊成一顆新積木,重複到湊夠積木箱大小。整個過程是
**由小到大、只加不減**——牌庫是一步步長出來的,而且每一步的合併規則都被記錄成一份有優先順序的
清單,推論時照這份清單貪婪套用,同一句話永遠切出同一種結果。

## Unigram(SentencePiece)是「篩選」哲學:反過來做

Unigram 子詞分割演算法(常被裝在 **SentencePiece** 這個工具包裡使用)整個方向反過來:

1. **先列一大堆候選**:從語料裡收集大量候選子詞(可以是常見字元組合、常見子字串,起始候選數量
   遠大於最終目標)。
2. **用機率模型幫每個候選打分**:把牌庫當成一個 unigram 語言模型——假設每個 token 出現互相獨立,
   每個候選子詞都有一個機率,用 EM(Expectation-Maximization)演算法反覆估計這些機率,讓「用
   目前這套候選牌庫去切整個語料庫」的整體概似度(likelihood)最大化。
3. **反覆砍牌**:每一輪,算出「拿掉某個候選之後,整個語料庫的概似度掉多少」——掉得最少的那些
   候選,代表它們對牌庫的貢獻最小,就砍掉。重複「估計機率 → 砍掉貢獻最小的一批」,直到牌庫縮到
   目標張數為止。

跟 BPE 最大的不同,不只是「合併 vs 砍」的方向相反,還有一個結構性差異:**BPE 推論時照固定優先
順序貪婪套用,同一句話永遠只有一種切法;Unigram 的牌庫是一個機率模型,同一個詞常常有好幾種合法
切法,每種切法有自己的機率**——例如英文 `unwanted` 理論上可以切成 `un` + `want` + `ed`,也可以
切成 `un` + `wanted`,兩種切法在 Unigram 的框架下都合法,只是機率不同。推論時通常還是選機率
最高(或用動態規劃求最佳)的那一種,但這個「多種合法切法各自帶機率」的性質,被拿去做了一個額外
用途:**subword regularization**——訓練時故意按機率隨機抽一種切法而不是永遠用最佳切法,讓模型
看到同一個詞的不同切法版本,增加穩健度。這是 Unigram 演算法論文本身提出的訓練技巧,不是切詞
機制的必要部分。

**兩種哲學對照**:

| | BPE(合併) | Unigram(篩選) |
|---|---|---|
| 方向 | 由小到大,只加 | 由大到小,只減 |
| 判斷依據 | 相鄰配對出現頻率 | 對整個語料庫概似度的貢獻 |
| 推論時的切法 | 固定優先順序,貪婪套用,結果唯一 | 機率模型,同一個詞可能有多種合法切法 |
| 額外用途 | 無 | 可做 subword regularization(訓練時隨機抽切法) |

兩者都是解決同一個問題(積木箱多大、怎麼切最合理)的不同手段,實務上常被放在同一個工具包
(SentencePiece)底下,依模型設計選其中一種。

## Weight tying:查牌櫃跟打分,常常是同一張表

[`tokenizer-bpe.md`](2026-07-04-tokenizer-bpe.md) 最後一節提過兩個動作:模型輸入端要把每個
字牌編號換成一條向量(查「牌櫃收納位置」,也就是 embedding),模型輸出端要對牌庫裡每一張牌
打一個分數(next-token prediction)。這兩個動作各自對應一張表:

- **輸入端**:一張「編號 → 向量」的表,形狀是「牌庫張數 × 向量維度」。
- **輸出端**:一張「向量 → 每張牌的分數」的表,形狀是「向量維度 × 牌庫張數」——剛好是輸入端
  那張表的**轉置**。

兩張表形狀恰好互為轉置,很多模型乾脆讓它們**共用同一份參數**——輸出端的打分矩陣直接拿輸入端
embedding 表的轉置來用,不另外學一組獨立參數。這個做法叫 **weight tying(權重綁定)**,原始
提案來自兩篇獨立發表、結論一致的論文(Press & Wolf, 2017;Inan et al., 2017)。理由有兩個:

1. **省參數**:牌庫張數乘以向量維度往往是模型裡最大的一塊參數量之一(牌庫幾萬張 × 向量維度
   往往上千),兩張表變一張,省下的參數量相當可觀。
2. **概念上說得通**:兩個動作其實是同一件事的正反面——「查牌櫃收納位置」是「給編號找向量」,
   「打分」是「給向量找最像哪個編號」,方向相反但用的是同一套「編號 ↔ 位置」的對應關係,共用
   一份參數等於強迫模型把這套對應關係學得更一致。

GPT-2 是採用 weight tying 的例子之一。

## 誠實提醒:不是所有模型都綁

**這是一個設計選擇,不是規則**——決定綁不綁,要看那個模型自己的設計文件或原始碼,不能預設。
一個值得記住的走向:牌庫張數固定的情況下,模型整體規模越大,這張共用表佔全部參數的比例就越小,
「省參數」的誘因跟著變小;某些近期公開的大模型技術報告顯示,不綁(輸入輸出各自一張獨立表)在
更大規模下反而更常見——這讓輸出端的表有機會學出跟輸入端不同、更適合「打分」這個任務的形狀。
但這不是絕對規律,判斷任何一個具體模型有沒有綁,還是要查那個模型自己的說明。

## 出處

- [Kudo & Richardson, 2018, *SentencePiece: A Simple and Language Independent Subword Tokenizer
  and Detokenizer for Neural Text Processing*](https://aclanthology.org/D18-2012/)(EMNLP 2018
  System Demonstrations)——SentencePiece 工具包論文,說明它同時支援 BPE 與 Unigram 兩種演算法、
  能直接吃原始句子不需要預先按空白斷詞。VERIFIED(標題、作者、發表會議、核心設計動機經 WebSearch
  交叉確認)。
- [Kudo, 2018, *Subword Regularization: Improving Neural Network Translation Models with Multiple
  Subword Candidates*](https://aclanthology.org/P18-1007/)(ACL 2018)——Unigram 子詞分割演算法
  的原始論文:先列大量候選、用 EM 訓練 unigram 語言模型、反覆砍掉機率貢獻最低的候選直到湊夠
  目標張數;同時提出訓練時按機率隨機抽切法的 subword regularization 技巧。VERIFIED(標題、作者、
  發表會議、演算法流程經 WebSearch 交叉確認)。
- [Press & Wolf, 2017, *Using the Output Embedding to Improve Language Models*](https://aclanthology.org/E17-2025/)
  (EACL 2017)——weight tying 的原始提案之一,論證輸出端矩陣本身就是一種合法的詞向量,建議跟
  輸入端 embedding 綁在一起。VERIFIED(標題、作者、發表會議經 WebSearch 交叉確認)。
- [Inan, Khosravi & Socher, 2017, *Tying Word Vectors and Word Classifiers: A Loss Framework for
  Language Modeling*](https://arxiv.org/abs/1611.01462)(ICLR 2017)——另一篇獨立提出 weight
  tying 的論文,從損失函數理論框架推導出綁定輸入輸出矩陣的做法。VERIFIED(標題、作者、發表會議
  經 WebSearch 交叉確認)。
- GPT-2 採用 weight tying——LIKELY(經 WebSearch 查到多篇二手技術解說一致提及,未逐段核對
  GPT-2 原始論文或原始碼)。
- 近期部分大模型技術報告傾向不綁(輸入輸出各自獨立矩陣)——LIKELY(主要依據單一篇二手分析
  文章的整理說法,未逐一查證各模型自己的技術報告原文)。

**誠實聲明**:「造積木箱」「篩選 vs 合併」的哲學對照角度,是這篇筆記自己的歸納,論文原文沒有
用這組比喻;`unwanted` 切成 `un+want+ed` 或 `un+wanted` 的例子是自己編的示範,用來說明「同一個
詞多種合法切法」這個性質,不是任何論文裡的具體數字。「輸入輸出兩張表互為轉置」的形狀描述是
標準線性代數事實(embedding 表 `V×d`,輸出投影自然是 `d×V`),不是單一論文的主張。
