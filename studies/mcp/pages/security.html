101 之外的進階主題。圖 A 是責任怎麼分兩邊,圖 B 是整篇最重要的一張圖——三個能力疊在一起 才是真正的危險,圖 C 是遠端授權(OAuth 2.1)實際怎麼跑,圖 D 是四個官方文件實際列出的 具體攻擊。點任何卡片看細節。

圖 A · 誰該防誰
責任分兩邊,少了任何一邊都補不回來

Server 負責不要輸出髒東西,Client 負責不要盲目相信輸入的東西——這是官方規格對 Tools 寫的安全考量,雙邊各有一份清單。

● Server MUST

驗證所有工具輸入

Validate all tool inputs

實作存取控制

Implement proper access controls

限流

Rate limit tool invocations

清理輸出

Sanitize tool outputs
● Client SHOULD

敏感操作前要求確認

Prompt for confirmation

執行前顯示輸入內容

Display tool inputs before execution

結果送進 LLM 前再驗一次

Validate tool results before passing to an LLM

逾時 + 留稽核紀錄

Implement timeouts, maintain logs

圖 C · 遠端授權怎麼跑
OAuth 2.1 — MCP server 只是「資源」,不是「授權者」

前面的 Confused Deputy、Token Passthrough,都是這套流程沒做好才長出來的問題。關鍵分工: MCP server 是被保護的資源(Resource Server),不管登入、不管同意畫面——那些是外部授權 伺服器的事。想成一個音樂節戴手環的流程:點場景裡的任一個角色,或下面任一步,兩邊 會同步高亮。
