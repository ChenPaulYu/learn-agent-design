圖 C · Programmatic Tool Calling(Code Mode)
多輪來回,壓成一次

切換下面兩個按鈕,看同樣要呼叫三個工具的任務,「一般呼叫」跟「Code Mode」的流程差在哪裡。

一般呼叫 — 每次都要回來問模型
Code Mode — 寫一次程式碼,沙箱裡跑完

跟 tools/list 的 pagination 不是同一件事

cursor/nextCursor 是單一 server 工具清單太長時,線路層的分頁——防的是「一次傳輸塞爆」,清單最終還是全部到 client 手上。

Progressive Discovery 管的是另一層

管的是「host 決定要不要把某個工具的完整定義塞進模型的 context」——這是模型能看到多少的問題,跟「傳輸怎麼分批」是兩層不同的關注點,容易被搞混。
