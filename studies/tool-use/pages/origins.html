圖 B · 是誰讓土法煉鋼時代結束的
OpenAI 把「各自發明暗號」收編成「官方統一表單」

圖 D · 不等官方機制的案例
Perplexity:概念比官方機制早,產品也比官方機制早

跟 WebGPT/MRKL 那類學術先驅同一種故事,只是這次是拿去做成真正上線的消費產品。

2022-12-07 · Perplexity Ask 上線
比 OpenAI 官方 Function Calling(2023-06)還早半年——用的是自己客製的 RAG pipeline (LLM + 自製搜尋/檢索),完全不依賴任何廠商的官方 tool-use API。不用等廠商把機制做成 標準,自己就能土法煉鋼湊出一個堪用的版本先上線。 LIKELY — 多個第三方來源一致,未直接抓到 2022 年當下的官方原文
圖 E · 模型怎麼「學會」格式,走的是三段式弧線
窄且訓練 → 廣但要教 → 廣且訓練

容易漏掉的地方:ReAct 不是「訓練 vs prompt」這條線的起點——WebGPT 更早,而且本身就是 訓練出來的,只是很窄(只認瀏覽器)。切換三個時代看差在哪。

WebGPT · 窄且訓練
ReAct · 廣但要教
現在 · 廣且訓練
但有一個地方不能講得太滿
格式不用教了,不代表完全不用給任何輸入——「宣告有哪些工具可以用」這件事還是要做, `tools` 參數(name/description/schema)還是得送進 API 請求裡,只是這不算傳統意義的 prompt(不是自由文字、不是 few-shot 範例),是結構化的 API 參數。
誠實聲明 — Anthropic/OpenAI/Google 都沒有公開揭露訓練 tool use 的確切配方,這是研究社群的方法論共識,不是廠商官方揭露
