業界對照 · 獨創
OpenAI 的官方指引是把範例寫進 description 自由文字裡,沒有獨立的 examples 欄位;Google Vertex AI 的 FunctionDeclaration schema 支援 屬性清單裡明講沒有 example 這個關鍵字,官方建議一樣是塞進 description。查證當下沒找到業界通用叫法,也沒找到平行案例。
跟 Agent Skills 的邊界 · 同精神,顆粒度差一個量級
沒有業界平行案例,不代表這個「給範例」的手法在 Anthropic 自己的產品線裡是孤例—— Agent Skills 的 SKILL.md「## Examples」段落 + examples.md 附屬檔案,解決的是同一種精神的問題。差別是顆粒度差一個量級:這裡的 input_examples 只到「一次函式呼叫的參數」,是工具定義的一部分,每次都 跟著送;Skills 的範例範圍是「一整個任務/流程」,是 progressive disclosure 最後一層, 判斷需要才讀。完整比較在 agent-skills 主題的 101 頁。
Tool Use Examples — 範圍只到「這一張表單怎麼填」: { "name": "create_ticket", "input_examples": [ {"title": "Login page returns 500 error", "priority": "critical", "labels": ["bug", "authentication", "production"], "reporter": {"id": "USR-12345", "name": "Jane Smith"}, "escalation": {"level": 2, "notify_manager": true, "sla_hours": 4}}, {"title": "Add dark mode support", "labels": ["feature-request", "ui"]}, {"title": "Update API documentation"} ] } // 範例就是「表單填法」本身——三組範例,示範同一張表單的不同填寫深度
Skills 的 Examples — 範圍是「這整件事怎麼做完」: --- name: pdf-processing description: Extract text and tables from PDF files, fill forms, merge documents... --- ## Instructions [怎麼判斷用哪個函式庫、怎麼處理掃描檔跟原生 PDF 的差異...] ## Examples [示範完整流程:收到一份 PDF → 判斷類型 → 抽取表格 → 填回原表單欄位 → 輸出] // 範例是「一整條任務怎麼跑完」——不是某一次函式呼叫的參數怎麼填
