Agent 解剖學 — 五層、那條沒名字的邊、探索政策

五層,各自回答一個問題

Prompt 不是一層 — 它是 primitive 的 instantiation(具體內容),不是 primitive 本身。
Multi-Agent 不是 Computation Model 的取值 — 它是好幾個「各自完整」的 agent 堆疊怎麼組合,單位跟「一個決策者怎麼探索」不同,層級不一樣。

圖 C · 最容易講糊的一層
Computation Model = 探索政策,不是形狀

慣例把這層講成「execution topology(線/樹/圖)」——但那只是結果的幾何描述。 First-principles 重建後:它是一個「計算預算怎麼分配去探索 Pattern 打開的候選延續空間」的 政策,topology 只是政策跑出來的痕跡。每個政策由四要素組成,點下面任一個看拆解。

Linear

每步只算一條

Self-Consistency

N 條獨立解投票

Beam Search

每步固定留 K 條

Tree of Thoughts

分岔+評分+剪枝

MCTS / LATS

模擬到底再回推

Graph of Thoughts

分岔可以合併回來

迭代修正(Reflexion 式)

跑完自我批評,決定重不重跑

遞迴分解(RLM 式)

不是探索候選,是拆子問題

↑ 點一個政策,看它的四要素:分岔幾條 / 什麼結構 / 怎麼比較 / 何時收斂。

陷阱:prompt 裝成 Computation Model — 寫一個 prompt 叫模型「自己在心裡想三種做法、選一個」,全部在同一次生成裡完成——這是 Pattern 的內容在裝成探索,沒有真的多分配運算資源,不是 Computation Model。CM 永遠是 code(控制流程),不是 prompt。
Debate 類要看實作 — 同一個決策迴路生兩個候選角色互相比較,還算 CM;若是兩個各自獨立完整的 agent 在對話,單位不對,跟 Multi-Agent 一樣不屬於這層。
