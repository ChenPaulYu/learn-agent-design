# CLAUDE.md — learn-agent-design

A personal learning workspace for agent-design concepts: read/reason about a concept, distill it
into a note, then try it hands-on in code. Not a product repo — no shipping, no users.

## The repo's operating rules live in `rules/`

Everything about *how this repo is organized and built* — folder layout, the `docs/`-is-a-thought
convention, how to build & wire an interactive `pages/*.html` file — is in
[`rules/_index.md`](rules/_index.md), not here. This file stays the short entry point (what the
repo is, where to look next); `rules/` is where the actual conventions live, split by concern
(`organization.md`, `pages.md`, more as they accumulate) so any one of them stays focused instead
of growing into one long, hard-to-navigate wall of rules.
